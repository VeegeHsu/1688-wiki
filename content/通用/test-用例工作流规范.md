# 测试用例工作流规范

## 规则 1：JSON 是唯一权威来源

始终编辑 JSON 文件，再用 Python 脚本重新生成 XMind，不直接修改 XMind 文件。

> **为什么**：直接修改 XMind 会导致来源和产物不一致，难以追溯。

**操作原则**：每次用例变更都先改 JSON，改完验证后再跑脚本生成 `.xmind`。

---

## 规则 2：用例纠错必须同步更新知识库

钉钉 Agent 测试用例修正后，必须同步更新对应的业务知识库文档，将错误描述和正确逻辑对照表写入文档。

> **为什么**：知识库是未来用例设计的参考来源，漏记会导致同类错误重复出现。

**操作原则**：每次优化用例结束时，检查知识库是否同步更新，未更新则补充。

---

## 规则 3：JSON 中文引号用【】替代

在 JSON 字符串值内部，用中文引号 `"文本"` 会导致 `JSONDecodeError`，必须用 `【文本】` 替代。

> **为什么**：JSON 解析器将双引号视为字符串结束符，中文引号同样被解析为 ASCII 双引号（`"`）。

**操作原则**：编写或审查测试步骤、期望结果等字段时，凡需强调词语一律用 `【】`，不用 `""`。
