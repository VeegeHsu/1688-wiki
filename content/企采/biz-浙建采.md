# 浙建采业务逻辑

## 对接模式

浙建采与 1688 企采采用 **SKU 模式**对接（区别于商越的 SPU 模式）：

```
1688 商品 N 个 SKU = 浙建采 N 个商品（1:1 映射）
```

## 核心业务规则

| 规则 | 说明 |
|------|------|
| SKU 映射 | 1688 商品每个 SKU 在浙建采侧对应一个独立商品 |
| 下架联动 | 浙建采下架 1 个商品 → 1688 侧**产品维度**下架 → 同步下架该产品所有 SKU |
| 大客户标识 | 浙建采默认 `isKa=1`（区别于普通渠道） |
| 地址库 | 需单独对接浙建采地址库 |
| 类目映射 | 需对接浙建采类目映射 |

## 价格体系

| 价格类型 | 计算方式 |
|----------|----------|
| 市场价 | 按正常 TR 上浮 11% |
| 官方专供价 | 与浙建采单独谈定的价格 |

## 对接分期

| 期次 | 内容 |
|------|------|
| 一期（铺品） | 渠道商品铺品链路、接口对接、账期结合、渠道订单发票、渠道订单部分优化 |
| 二期（正向） | 正向下单链路 |
| 三期（逆向） | 逆向退款链路 |
