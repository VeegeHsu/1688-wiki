# 企采渠道对接模式

## 渠道概览

| 渠道 | channel_code | 对接模式 |
|------|-------------|---------|
| 天猫 | `tmall` | 传统 API 对接 |
| 浙建采 | `zjc` | 传统 API 对接 |
| 飞猪 | `fliggy` | 企采 Agent 对接 |
| 钉钉 | `dingtalk` | 企采 Agent 对接 |

> 测试环境 channel_code：天猫用 `tmall_test`，浙建采用 `zjc_test`；用生产值查永远 0 行。

---

## 两大对接模式

### 传统 API 对接（天猫 + 浙建采）

**铺品流程**：
```
1688供应商报名企业采购自营大店
  ↓
生成企采产品 + 供货产品
  ↓
通过一审、二审
  ↓
渠道选品中心 → 定价 → 铺货到渠道
```

**订单履约链路**：渠道订单 → 履约单

**核心表**：

| 表名 | 说明 |
|------|------|
| `qycg_mall_channel_offer` | 渠道商品 |
| `qycg_mall_channel_sale_order` | 渠道订单主表 |
| `qycg_mall_channel_sale_order_detail` | 渠道订单明细 |
| `qycg_mall_fullfill_order` | 履约单 |

---

### 企采 Agent 对接（飞猪 + 钉钉）

**无需渠道铺品**，品池由 Agent 圈选召回。

**品池范围**：

| 渠道 | 品池 |
|------|------|
| 飞猪 | 仅企采自营品 |
| 钉钉 | 企采自营品 + 严选品 |

**订单履约链路**：采购单 → 合约单 → 1688 订单

**核心表**：

| 表名 | 说明 |
|------|------|
| `agent_purchase_order` | 采购单主表 |
| `agent_purchase_order_detail` | 采购单明细 |

---

## 关键名词

| 名词 | 定义 |
|------|------|
| **企采自营品** | 供应商报名自营大店 → 生成企采产品和供货产品 → 一审二审通过 → 定价上架 → 成为超链品 |
| **严选品** | 供应商商品满足特定条件 → 打严选标 → 成为严选货盘 |

---

## 测试重点

- 不同渠道的**商品来源**、**铺品流程**、**订单链路**完全不同，容易混淆。
- 测试前务必先确认 `channel_code`，再判断应该查哪张表、走哪条链路。
- 重点关注：铺品状态流转、一审二审、定价上架、品池圈选、跨链路订单一致性。
