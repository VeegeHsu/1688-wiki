# 飞猪 Agent 业务逻辑

## 支付链路与下单模式

| 支付方式 | 下单模式 |
|----------|----------|
| 储蓄罐支付 | 走**合约单**下单 |
| 支付宝支付 | 走**合约单**下单 |

> 注意与淘宝企业购的区别：淘宝企业购储蓄罐走超链一段单。详见 [[biz-淘宝企业购]]

## 飞猪账号体系

| 概念 | 说明 |
|------|------|
| 飞猪账户 | 主账户 |
| hid | 买家组织（门店 id），免房资金存在 hid 下 |
| 绑定关系 | 飞猪B账号 — 手机号 — 1688C账号 |

**换绑规则**：换绑后，飞猪B账号仍可查看原C账号的采购记录，但**不能操作退货**。

## 采购单状态流转

```
调用飞猪储蓄罐扣减成功 → 采购单变为"下单中"
监听 1688 订单已付款消息 → 采购单变为"采购中"
```

储蓄罐支付和支付宝支付，给飞猪的都是**采购单 id**。

## 储蓄罐并发风险（技术评审要点）

| 情况 | 采购单状态 | 处理方式 |
|------|-----------|----------|
| 飞猪未收到请求 | 维持原状 | 1688 需要重试 |
| 飞猪收到但未完成 | 下单中 | 等待 |
| 查询余额不足 | 下单中 | 保持，提示用户 |
| 余额通过但虚拟扣减失败 | 下单中 | 保持 |
| 扣减成功但飞猪打款失败 | 下单中 | **无取消行动点**（异常态） |

**状态跃迁风险**：飞猪通知打款成功超时，但实际已打款成功；若 1688 监听到订单已付款，直接从"下单中"跃迁到"采购中"。

## 逆向场景

- **售中退（储蓄罐支付方式）**：需调用飞猪的储蓄罐退回接口，通过采购单 id 发起

## 核心 mtop 接口

| 接口 | 说明 |
|------|------|
| `mtop.1688.qycg.agent.get.channel.can.pay.by.cxg` | 查询是否支持储蓄罐支付 |
| `mtop.1688.qycg.agent.query.cxg.balance` | 查询储蓄罐余额（返回 `useBalance` 单位为分） |
| `mtop.1688.qycg.agent.cxg.fliggy.pay` | 调用飞猪储蓄罐支付（整体入口） |
| `mtop.1688.qycg.agent.query.agent.offer.tab.for.mall` | 查询类目 tab（商品列表页） |

## 核心 HSF 接口

| 接口 | 说明 |
|------|------|
| `AgentCxgFacadeImpl#cxgFliggyPay` | 储蓄罐支付主流程 |
| `CorporatePaymentService~transfer` | 对公支付转账（关键日志：`bankTransferResultResult`） |
| `FliggyServiceMarketHotelService~payToHotel` | 通知飞猪打款（channel: `fliggy_hotel_cash_payment`） |
