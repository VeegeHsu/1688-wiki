# KA账期（背靠背金融保险账期）

## 背景

1688 企业采购背靠背金融保险账期产品，针对 **KA 大企业客户**（`isKa=1`）提供账期能力（2025-07 一期上线，2026-02 二期）。

核心机制：平台对买家提供账期，同时对供应商垫资，中间由金融保险产品承担风险。

## 额度体系

| 概念 | 说明 |
|------|------|
| 授信额度 | 平台给买家的总额度上限 |
| 预占额度 | 已提交但未完成结算的金额 |
| 占用额度 | 已结算但账期内未还款的金额 |
| 剩余可用额度 | 当前仍可使用的额度 |

**校验公式：**

```
授信额度 = 预占额度 + 占用额度 + 剩余可用额度
总公司额度 = ∑分公司下占用明细金额 + 剩余可用额度
```

## 并发防控

所有涉及额度变更的接口均对 `customerId` 加锁：

| 操作 | 锁粒度 |
|------|--------|
| 预占 / 占用接口 | `customerId` |
| 核销接口 | `customerId` |
| 账单创建 | `customerId` |

## 资损风险场景（测试重点）

| 风险类型 | 场景描述 |
|----------|----------|
| 额度扣减/释放有误 | 并发下预占/释放顺序错乱 |
| 账单漏单/重复 | 消息重试导致账单重复创建 |
| 金额单位有误 | 元/分单位混用 |
| 核销并发 | 多笔核销同时命中同一账单 |
| 关闭账期与使用额度并发 | 账期被关闭时仍有进行中的扣减 |

## 关键 DB 表

| 表名 | 唯一索引 | 说明 |
|------|----------|------|
| `qycg_mall_insurance_order` | 应收账单明细号 + 渠道订单明细行 | 保险账期订单记录 |
| `qycg_mall_receipt_info` | 流水号 | 收款流水记录 |

## 订单链路关键节点

| 节点 | 关键动作 |
|------|----------|
| 创建渠道订单 | 验证企采预占额度 |
| 接单前 | syt 占用 + 采购单创建（验金融额度占用成功） |
| 确收 | 预占转占用 + 同步 arm |
| 确收后 | 保险订单明细生成（等待投保状态）→ 生意通投保成功（init 状态，外部保单号已记录） |

确收关键日志/接口：`SytFacadeImpl#confirmSytPurchaseOrder`

确收相关 SQL：
```sql
SELECT * FROM qycg_mall_insurance_order WHERE relate_channel_sale_order_detail_code = '{子单号}';
```
